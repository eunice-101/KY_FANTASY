# 나만의 블로그 글쓰기 워크플로우 (My Blog Writing Workflow)

워드프레스 블로그에 최적화된 5단계 글쓰기 프로세스.

## 스타일 로딩

`my-style.md` 파일에서 사용자 스타일을 읽어온다. 없으면 기본 스타일 적용.

---

## 블로그 플랫폼 이해

### SEO 검색 유입 기반

워드프레스 기반 블로그의 핵심 트래픽은 **검색 엔진 유입**이다. 제목·소제목·본문에 검색 키워드가 자연스럽게 포함되어야 한다.

### 독자 행동 패턴

- 검색 → 제목+메타설명 확인 → 클릭 → TOC로 원하는 섹션 점프
- **소제목을 훑으며 필요한 부분만 읽는 독자**가 많다
- 읽기 시간: 3~7분 (1,200~2,500자)

### 블로그 글 기본 특징

**단락 구조:** 문단은 2~5문장이 하나의 의미 단위를 이루는 일반 문서 형식.
**내부 링크:** 관련 글·시리즈 글을 자연스럽게 본문에 삽입.

---

## 전체 흐름

```
Step 1: 주제·앵글 설계 → Step 2: 구조 설계 → Step 3: 초안 작성 → Step 4: 리라이팅 → Step 5: 발행 준비
```

---

## 기본 글쓰기 원칙

### 문체 기준 (my-style.md가 없을 때)
- 표준어 사용, '~합니다'체 고정
- 문장 15~25단어, 한 문장에 한 정보
- 문단 2~5문장, 일반 문서 형식
- 감정적 표현·감탄형 지양
- 차분하고 설명적인 문체

### 금지어 규칙
과장형 형용사, 모호한 부사, 추측·완곡 표현, AI 흔적 조합 — 모두 금지.

### 금지 요소
- 정보 나열식 구성, 감상형 표현, 과도한 반복, 과장
- "오늘은 ~에 대해 이야기해 보겠습니다" 식의 메타 서술
- 검색 키워드 부자연스러운 반복 삽입
- 문장별 단독 줄바꿈(모바일식 시적 배치)

## Step 1: 주제·앵글 설계

### 수집 항목

| 항목 | 질문 | 예시 |
|------|------|------|
| **주제** | 무엇에 대해 쓰는가? | AI 활용법, 프로그래밍 입문, 마케팅 전략 |
| **앵글** | 기존 관점과 무엇이 다른가? | "코딩의 핵심은 문법이 아니라 논리적 사고다" |
| **독자** | 누가 읽는가? | 입문자, 실무자, 학생, 직장인 |
| **글 유형** | 어떤 형식인가? | 방법론 해설 / 도구 리뷰 / 인사이트 / 가이드 |
| **카테고리** | 어떤 카테고리에 속하는가? | 사용자 블로그 카테고리 |
| **시리즈** | 기존 시리즈에 포함? | 시리즈명, 회차 번호 (선택) |

### SEO 키워드 설계

핵심 키워드 1개 + 보조 키워드 2~3개. 자연스러운 문맥 내 사용 우선.

---

## Step 2: 구조 설계

### 글 유형별 구조

**방법론 해설:**
```
도입: 독자의 문제 상황
H2: 개념 정의
H2: 단계별 설명
  H3: 각 단계의 판단 기준
H2: 주의사항 또는 오해 교정
마무리: 핵심 기준 압축
```

**도구·기능 리뷰:**
```
도입: 왜 이 도구를 검토했는가
H2: 핵심 기능과 평가 기준
  H3: 기능별 실험 결과
H2: 기존 도구와의 차이
H2: 어떤 상황에서 선택해야 하는가
마무리: 판단 기준 정리
```

**가이드·튜토리얼:**
```
도입: 이 가이드가 필요한 상황
H2: 사전 준비 / 전제 조건
H2: 단계별 실행
  H3: 각 단계 (예시, 스크린샷 위치 포함)
H2: 자주 발생하는 문제
마무리: 핵심 실행 기준 정리
```

### 소제목(H2/H3) 설계 규칙

- H2는 2~5개, H3는 H2당 0~3개
- **독자가 소제목만 읽어도 글의 골격을 파악**할 수 있어야 한다
- 기능명이나 명사 나열 금지 → 판단·행동 중심 문장

### 제목 설계

- SEO 핵심 키워드 자연스럽게 포함
- 50자 이내 권장
- 제목 후보 **5개** 생성

---

## Step 3: 초안 작성

### 초안 작성 규칙

- Step 2 구조 따라 작성
- `my-style.md` 스타일 적용
- **도입부 3~4문장에 특별히 집중** — 검색 결과 미리보기 구간
- 분량: 1,200~2,500자
- 소제목 H2는 `##`, H3는 `###`
- 이미지 위치: `[IMAGE: 설명]`
- 내부 링크: `[LINK: 관련글 제목]`

### 도입부 설계

| 방식 | 예시 | 효과 |
|------|------|------|
| 문제 제시 | "매번 같은 실수를 반복하고 있다면, 문제는 실력이 아닙니다." | 공감+호기심 |
| 질문→반전 | "코딩을 잘하려면 뭘 배워야 할까. 대부분 문법이라고 답하지만, 정답은 다릅니다." | 사고 자극 |
| 상황 제시 | "세 번째 수정 요청을 받았을 때, 문제는 글이 아니었습니다." | 경험 공감 |

**금지되는 도입:** 메타 서술, 사전식 정의, 뉴스 리드식

---

## Step 4: 리라이팅

### 점검 1: 도입부

첫 문장이 독자의 문제/질문으로 시작하는가, 메타설명으로 활용 가능한가 점검.

### 점검 2: 구조

H2/H3 소제목 구조, 논리적 흐름, TOC 명확성 점검.

### 점검 3: 스타일

**`my-style.md` 설정과 일치하는지 점검.** 어조, 좋아하는/싫어하는 문장 스타일, 나만의 무기 활용, 금기 사항 미위반.

### 점검 4: 블로그 최적화

분량(1,200~2,500자), SEO 키워드, 내부 링크, 제목 매력도 점검.

---

## Step 5: 발행 준비

### 최종 점검 항목

| # | 점검 항목 | 기준 |
|---|----------|------|
| 1 | 제목 확정 | 후보 5개, SEO 키워드 포함, 50자 이내 |
| 2 | 맞춤법·문법 | 한국어 맞춤법, 띄어쓰기, 외래어 표기법 |
| 3 | 카테고리·태그 | 카테고리 1~2개, 태그 2~4개 |
| 4 | 메타설명 | 150자 이내, 핵심 키워드 포함 |
| 5 | 대표 이미지 방향 | 이미지 콘셉트 제안 |
| 6 | 분량·마커 확인 | 범위 내, [TODO] 해결 |
| 7 | 스타일 최종 체크 | `my-style.md` 설정과 최종 대조 |

### 산출물

| 파일 | 설명 |
|------|------|
| `blog-final.md` | 최종 완성본 (에디터 복사·붙여넣기용) |
| `blog-final.docx` | 문서 보관용 (요청 시) |

```
**워드프레스 발행 체크리스트:**
□ 제목
□ 카테고리: _____
□ 태그: _____, _____, _____
□ 메타설명 (150자 이내)
□ 대표 이미지
□ 내부 링크 확인
□ H2/H3 → TOC 확인
□ 발행
```
