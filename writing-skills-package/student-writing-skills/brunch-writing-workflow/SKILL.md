# 나만의 브런치 글쓰기 워크플로우 (My Brunch Writing Workflow)

브런치 플랫폼에 최적화된 5단계 글쓰기 프로세스.

## 스타일 로딩

`my-style.md` 파일에서 사용자 스타일을 읽어온다. 없으면 기본 스타일 적용.

---

## 브런치 플랫폼 이해

### 큐레이션 기반 플랫폼

브런치는 SEO 검색이 아니라 **에디터 큐레이션과 독자 반응 기반**으로 글이 노출된다.

### 에디터가 주목하는 요소

- 깊이 있는 통찰, 독자 문제 해결, 감성적 공감대, 니치 전문성, 시대적 흐름 반영

### 독자 행동 패턴

- 제목 **0.3초**, 도입부 **4.4초**, 최대 읽기 시간 약 **3분** (2,000자 이내)
- 비계획 구매자 — 검색이 아닌 추천·노출로 유입

### 키워드 특성

브런치 키워드는 SEO용이 아니다. 발행 시 키워드 3개를 선택하며, **독자가 반응하는 감정적 키워드**에 집중한다.

---

## 전체 흐름

```
Step 1: 주제·앵글 설계 → Step 2: 구조 설계 → Step 3: 초안 작성 → Step 4: 리라이팅 → Step 5: 발행 준비
```

---

## 기본 글쓰기 원칙

### 문체 기준 (my-style.md가 없을 때)
- 표준어, '~다'체 기본 (브런치 주류 문체)
- 문장 15~22단어 (모바일 가독성)
- 문단 2~4문장 (모바일 화면 고려)
- 차분하고 개인적인 문체

### 금지어 규칙
과장형 형용사, 모호한 부사, 추측·완곡 표현, AI 흔적 조합 — 모두 금지.
단, 성찰적 불확실성("~일지도 모른다")은 일상 관찰·성찰 유형에서 허용.

### 금지 요소
- 정보 나열식 구성, 감상형 표현, 과장
- "오늘은 ~에 대해" 식의 메타 서술
- 제목에 넘버링(1화, 2화) 또는 날짜

---

## Step 1: 주제·앵글 설계

### 수집 항목

| 항목 | 질문 | 예시 |
|------|------|------|
| **주제** | 무엇에 대해 쓰는가? | AI 활용, 직장 경험, 일상 관찰 |
| **앵글** | 독자가 몰랐던 것은? | "실패의 원인은 능력이 아니라 질문의 방향이었다" |
| **독자** | 누가 읽는가? | 입문자, 실무자, 같은 경험을 한 사람 |
| **글 유형** | 어떤 형식인가? | 경험 기반 / 전문 해설 / 일상 성찰 / 가이드 / 트렌드 |
| **매거진** | 기존 매거진에 포함? | 매거진명, 연재 순서 (선택) |

### 앵글 설계 기준

- **"그래서 뭐?"를 넘기는가** — 정보가 아닌 시선의 전환
- **나만 할 수 있는 이야기인가** — 경험, 전문성, 독자적 해석
- **지금 읽어야 하는 이유가 있는가** — 시의성, 독자와의 접점

---

## Step 2: 구조 설계

### 브런치 글 구조 패턴

**경험 기반 인사이트:**
```
도입: 구체적 장면 (공감)
전개 1: 발견한 문제 또는 오해
전개 2: 기준 재정의
마무리: 관점 전환 + 여운
```

**전문 분야 해설:**
```
도입: 독자의 문제 상황 또는 질문
본론 1: 핵심 개념 + 판단 기준
본론 2: 적용 방식 + 예시
마무리: 다음 행동 기준 정리
```

**일상 관찰·성찰:**
```
도입: 감각적 장면
전개: 관찰 → 생각 → 기존 생각과의 차이
마무리: 여운 — 질문 또는 장면 회귀
```

### 제목 설계

- **20자 이내** (브런치 상한)
- **앞 10자에 핵심** (모바일 잘림 대비)
- 넘버링·날짜 금지
- 제목 후보 **5개** + 부제

| 유형 | 예시 |
|------|------|
| 질문형 | "그 도구, 정말 써야 할까" |
| 상황 제시형 | "퇴사 후 첫 월요일" |
| 기준 제시형 | "좋은 글의 조건은 하나다" |
| 반전형 | "열심히 한 사람이 지는 이유" |
| 감각형 | "커피가 식는 동안" |

---

## Step 3: 초안 작성

### 초안 작성 규칙

- 분량: **1,200~2,000자** (3분 읽기)
- **도입부 3문장에 특별히 집중** (4.4초 규칙)
- 문단 2~4문장 (모바일 가독성)
- 소제목 2~3개 (1,500자 이하면 구분선 처리)
- `my-style.md` 스타일 적용

### 도입부 설계 (4.4초 규칙)

| 방식 | 예시 | 효과 |
|------|------|------|
| 장면 시작 | "회의실 불이 꺼지고, 프로젝터만 켜져 있었다." | 몰입 |
| 문제 직면 | "세 번째 수정 요청 때, 문제는 글이 아니었다." | 긴장 |
| 질문 | "왜 매번 같은 방식으로 실패하는 걸까." | 사고 자극 |
| 반직관 | "좋은 글은 잘 쓴 글이 아니다." | 호기심 |

**금지:** 메타 서술, 사전식 정의, 뉴스 리드식 도입

---

## Step 4: 리라이팅

### 점검 1: 도입부 (4.4초 규칙)

첫 문장이 장면·문제·질문·반직관 중 하나로 시작하는가, 미리보기에서 매력적인가.

### 점검 2: 구조

논리적 흐름, 소제목 행동·판단 중심, 마무리 관점 재정렬.

### 점검 3: 스타일

**`my-style.md` 설정과 일치하는지 점검.** 어조, 문장 스타일, 나만의 무기 활용, 금기 사항 미위반.

### 점검 4: 브런치 최적화

분량(1,200~2,000자), 모바일 가독성(문단 2~4문장), 제목(20자, 앞 10자 핵심), 톤 적합성.

---

## Step 5: 발행 준비

### 최종 점검 항목

| # | 점검 항목 | 기준 |
|---|----------|------|
| 1 | 제목 확정 | 20자 이내, 앞 10자 핵심 |
| 2 | 부제 확정 | 제목 보충 1문장 |
| 3 | 맞춤법·문법 | 한국어 맞춤법, 띄어쓰기 |
| 4 | 키워드 3개 | 감정적·주제적 키워드 조합 |
| 5 | 커버 이미지 방향 | 톤에 맞는 이미지 제안 |
| 6 | 분량 확인 | 1,200~2,000자(±20%) |
| 7 | 스타일 최종 체크 | `my-style.md` 설정과 최종 대조 |

### 산출물

| 파일 | 설명 |
|------|------|
| `brunch-final.md` | 최종 완성본 (브런치 에디터 복사·붙여넣기용) |
| `brunch-final.docx` | 문서 보관용 (요청 시) |

```
**브런치 발행 체크리스트:**
□ 제목 (20자 이내, 앞 10자 핵심)
□ 부제
□ 키워드 3개: _____, _____, _____
□ 커버 이미지
□ 매거진 선택 (해당 시)
□ 발행
```

| 단계 | 산출물 | 형태 |
|------|--------|------|
| Step 1 | 주제·앵글·독자·유형·매거진 요약 | 대화창 텍스트 |
| Step 2 | 아웃라인 + 제목 후보 5개 + 부제 | 대화창 텍스트 |
| Step 3 | 초안 | 대화창 텍스트 |
| Step 4 | 수정본 + 변경 요약 | 대화창 텍스트 |
| Step 5 | 최종본 + 발행 체크리스트 | `brunch-final.md` / `.docx` |
